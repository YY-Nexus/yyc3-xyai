#### 设计Token使用指南

##### 在Figma中使用

1. **创建Variables**: 在Figma中使用Variables功能创建上述所有Token
2. **应用Token**: 在设计元素中直接引用Variables，而不是硬编码值
3. **维护一致性**: 所有组件使用相同的Token，确保视觉一致性
4. **批量更新**: 修改Token值，所有引用该Token的组件自动更新

##### 在代码中使用

```css
/* CSS中使用 */
.button-primary {
  background-color: var(--primary-500);
  color: white;
  padding: var(--spacing-4) var(--spacing-6);
  border-radius: var(--radius-lg);
  font-size: var(--text-base);
  font-weight: var(--font-semibold);
  box-shadow: var(--shadow-md);
}

/* Tailwind CSS中使用 */
<button class="bg-primary-500 text-white px-4 py-6 rounded-lg text-base font-semibold shadow-md">
  主要按钮
</button>
```

##### Token命名规范

- 使用语义化命名，如 `primary-500` 而不是 `purple-500`
- 使用层级结构，如 `primary-50` 到 `primary-900`
- 使用功能性命名，如 `success`、`warning`、`error`
- 遵循 kebab-case 命名规范

#### 组件使用建议

1. **按钮组件**：
   - 主要操作使用主要按钮
   - 次要操作使用次要按钮
   - 链接操作使用文本按钮
   - 图标操作使用图标按钮

2. **输入框组件**：
   - 短文本使用文本输入框
   - 搜索使用搜索输入框
   - 长文本使用多行输入框
   - 始终提供清晰的占位符

3. **卡片组件**：
   - 内容分组使用基础卡片
   - 功能展示使用功能卡片
   - 信息展示使用带标题卡片
   - 保持卡片内容简洁

4. **标签组件**：
   - 状态标识使用语义标签
   - 分类标签使用默认标签
   - 避免使用过多标签
   - 保持标签文字简洁

5. **导航组件**：
   - 桌面端使用顶部导航
   - 复杂应用使用侧边导航
   - 移动端使用底部导航
   - 保持导航层级清晰

6. **AI浮窗组件**：
   - 默认状态为收起状态
   - 用户交互时展开
   - 支持最小化和关闭
   - 保持浮窗位置合理

7. **消息气泡组件**：
   - 用户消息右对齐
   - AI消息左对齐
   - 区分不同消息类型
   - 提供时间戳

8. **加载状态组件**：
   - 内容加载使用骨架屏
   - 操作加载使用加载指示器
   - 提供加载提示文字
   - 保持加载动画流畅

9. **提示组件**：
   - 成功操作使用成功提示
   - 错误操作使用错误提示
   - 警告操作使用警告提示
   - 信息操作使用信息提示

10. **模态框组件**：
    - 重要操作使用模态框
    - 提供清晰的标题和内容
    - 提供确认和取消按钮
    - 支持点击遮罩关闭

**接口分层架构**：

```
API Gateway Layer → Business Layer → Data Access Layer
```

**Figma设计映射**：

- **API Gateway Layer**：设计统一的API请求状态展示（Loading、Success、Error）
- **Business Layer**：设计各业务模块的UI组件（用户、内容、AI、成长）
- **Data Access Layer**：设计数据加载、缓存、错误处理的UI反馈

#### 2. **接口版本管理与UI设计**

**版本管理策略**：

- v1版本：稳定版本，保持向后兼容
- v2版本：新功能版本，逐步替代v1
- 版本切换：客户端指定、默认版本、灰度发布、回滚机制

**Figma设计要点**：

- 设计版本切换UI组件（设置页面）
- 设计版本提示信息（新功能引导）
- 设计版本兼容性提示（降级提示）

#### 3. **用户服务接口与UI设计**

**用户服务接口**：

- 获取用户信息：`GET /api/v1/users/{userId}`
- 更新用户信息：`PUT /api/v1/users/{userId}`
- 获取用户偏好：`GET /api/v1/users/{userId}/preferences`
- 更新用户偏好：`PUT /api/v1/users/{userId}/preferences`

**Figma设计要点**：

- 设计用户信息展示页面（头像、昵称、成长阶段）
- 设计用户信息编辑页面（表单、验证、保存）
- 设计用户偏好设置页面（主题、语言、通知）
- 设计用户成长记录展示页面（成长曲线、里程碑）

#### 4. **内容服务接口与UI设计**

**内容服务接口**：

- 获取内容列表：`GET /api/v1/contents`
- 获取内容详情：`GET /api/v1/contents/{contentId}`
- 内容搜索：`GET /api/v1/contents/search`
- 内容推荐：`GET /api/v1/contents/recommend`

**Figma设计要点**：

- 设计内容列表页面（卡片、列表、瀑布流）
- 设计内容详情页面（标题、正文、媒体、标签）
- 设计内容搜索页面（搜索框、筛选器、结果展示）

#### 5. **AI服务接口与UI设计**

**AI服务接口**：

- 发送文本消息：`POST /api/v1/ai/text-message`
- 开始语音识别：`POST /api/v1/ai/voice-recognition/start`
- 停止语音识别：`POST /api/v1/ai/voice-recognition/stop`
- 获取AI回复：`GET /api/v1/ai/response`
- 获取对话历史：`GET /api/v1/ai/conversation-history`

**Figma设计要点**：

- 设计AI浮窗交互界面（文本输入、语音输入、AI回复展示）
- 设计语音识别状态UI（录音中、识别中、识别完成）
- 设计对话历史页面（时间线、消息气泡、操作按钮）

**Figma设计要点**：

### 接口设计原则与UI设计规范

**接口设计原则**：

- RESTful风格：资源导向的URL设计
- 统一响应格式：成功、失败、错误码
- 分页查询：支持大数据量展示
- 异步处理：长时间操作的状态反馈

**Figma设计规范**：

- 设计统一的Loading状态（骨架屏、加载动画）
- 设计统一的Error状态（错误提示、重试按钮）
- 设计统一的Empty状态（空数据提示、引导操作）
- 设计统一的Success状态（成功提示、操作反馈）

### 接口安全与性能与UI设计

**接口安全机制**：

- JWT认证：Token过期提示、重新登录引导
- HTTPS：安全连接提示、证书验证UI
- RBAC权限：权限不足提示、申请权限UI

**性能优化策略**：

- 缓存策略：缓存命中提示、缓存失效处理
- 分页查询：分页器、加载更多、无限滚动
- 异步处理：任务进度条、后台任务通知

**Figma设计要点**：

- 设计Token过期提示弹窗（重新登录引导）
- 设计权限不足提示页面（权限说明、申请按钮）
- 设计缓存状态指示器（缓存命中、缓存失效）
- 设计分页器组件（页码、跳转、每页数量）
- 设计任务进度条（进度百分比、剩余时间、取消按钮）

### 接口监控与日志与UI设计

**接口监控指标**：

## 🎯 接口架构设计文档在Figma设计中的实际应用案例

### 案例1：用户信息页面设计

**接口参考**：

- 获取用户信息：`GET /api/v1/users/{userId}`
- 更新用户信息：`PUT /api/v1/users/{userId}`

**Figma设计要点**：

1. **页面布局**：顶部头像、昵称、成长阶段；中间用户信息表单；底部保存按钮
2. **状态设计**：
   - Loading状态：骨架屏展示用户信息区域
   - Success状态：用户信息正常展示，保存成功提示
   - Error状态：错误提示、重试按钮
   - Empty状态：用户信息为空时的引导提示
3. **交互设计**：
   - 点击编辑按钮进入编辑模式
   - 表单验证（必填项、格式验证）
   - 保存时显示Loading状态
   - 保存成功显示成功提示
   - 保存失败显示错误提示和重试按钮

### 案例2：AI浮窗交互界面设计

**接口参考**：

- 发送文本消息：`POST /api/v1/ai/text-message`
- 开始语音识别：`POST /api/v1/ai/voice-recognition/start`
- 停止语音识别：`POST /api/v1/ai/voice-recognition/stop`
- 获取AI回复：`GET /api/v1/ai/response`

**Figma设计要点**：

1. **浮窗布局**：顶部AI头像、昵称；中间对话区域；底部输入区域
2. **状态设计**：
   - 语音识别状态：录音中（波形动画）、识别中（加载动画）、识别完成（文本展示）
   - AI回复状态：思考中（加载动画）、回复完成（文本展示）、回复失败（错误提示）
3. **交互设计**：
   - 点击麦克风按钮开始语音识别
   - 语音识别过程中显示波形动画
   - 识别完成后显示文本
   - AI回复时显示加载动画
   - 回复完成后显示文本
   - 支持文本输入和语音输入切换

### 案例3：成长记录页面设计

**接口参考**：

- 获取成长记录：`GET /api/v1/growth/records`
- 获取成长阶段：`GET /api/v1/growth/stages`
- 获取成长统计：`GET /api/v1/growth/statistics`

**Figma设计要点**：

1. **页面布局**：顶部成长统计卡片；中间成长阶段时间线；底部成长记录列表
2. **状态设计**：
   - Loading状态：骨架屏展示成长统计、阶段、记录区域
   - Success状态：成长统计、阶段、记录正常展示
   - Error状态：错误提示、重试按钮
   - Empty状态：暂无成长记录时的引导提示
3. **交互设计**：
   - 点击成长阶段卡片查看详细信息
   - 点击成长记录查看详情
   - 支持筛选成长记录类型
   - 支持按时间排序
