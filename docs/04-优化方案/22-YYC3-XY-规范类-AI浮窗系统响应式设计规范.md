## 🎯 设计目标

确保AI浮窗系统在不同设备上都能提供良好的用户体验，包括移动设备、平板、笔记本、台式机和大屏显示器。

## 📐 响应式断点定义

### 断点标准

| 断点名称 | 屏幕宽度 | 设备类型 | 典型设备 |
|----------|----------|----------|----------|
| xs | < 640px | 小型手机 | iPhone SE, 小屏Android手机 |
| sm | 640px - 768px | 大型手机 | iPhone 12/13/14, 大屏Android手机 |
| md | 768px - 1024px | 平板 | iPad, Android平板 |
| lg | 1024px - 1280px | 笔记本/小屏台式机 | MacBook Air, 小屏显示器 |
| xl | 1280px - 1536px | 台式机 | MacBook Pro, 标准显示器 |
| 2xl | > 1536px | 大屏显示器 | 4K显示器, 超宽屏 |

### 断点使用规范

```css
/* Tailwind CSS断点 */
@media (min-width: 640px) { /* sm */ }
@media (min-width: 768px) { /* md */ }
@media (min-width: 1024px) { /* lg */ }
@media (min-width: 1280px) { /* xl */ }
@media (min-width: 1536px) { /* 2xl */ }
```

## 🎨 AI浮窗响应式设计

### 浮窗尺寸规范

| 断点 | 宽度 | 高度 | 最大高度 | 定位 |
|------|------|------|----------|------|
| xs | 90vw | 60vh | 80vh | 居中 |
| sm | 85vw | 65vh | 85vh | 居中 |
| md | 75vw | 70vh | 85vh | 居中 |
| lg | 600px | 600px | 80vh | 右下角 |
| xl | 600px | 650px | 80vh | 右下角 |
| 2xl | 650px | 700px | 80vh | 右下角 |

### 浮窗定位策略

#### 移动设备（xs, sm）
- **定位**: 居中显示
- **行为**: 全屏模式，不可拖拽
- **原因**: 移动设备屏幕空间有限，居中显示提供最佳体验

#### 平板设备（md）
- **定位**: 居中显示
- **行为**: 可拖拽，限制在屏幕范围内
- **原因**: 平板设备屏幕适中，居中显示便于操作

#### 桌面设备（lg, xl, 2xl）
- **定位**: 右下角
- **行为**: 可拖拽，可最小化
- **原因**: 桌面设备屏幕空间充足，右下角定位不遮挡主要内容

## 🎯 适配策略

### 布局适配

1. **Flexbox布局**: 使用flex布局实现自适应
2. **Grid布局**: 复杂布局使用grid实现
3. **百分比宽度**: 使用百分比而非固定像素
4. **最大/最小宽度**: 设置合理的max-width和min-width

### 字体适配

1. **相对单位**: 使用rem或em而非px
2. **媒体查询**: 根据断点调整字体大小
3. **行高调整**: 保持合适的行高比例

### 图片适配

1. **响应式图片**: 使用srcset和sizes属性
2. **懒加载**: 大图片使用懒加载
3. **压缩优化**: 根据设备加载合适尺寸的图片

### 触摸适配

1. **触摸目标**: 最小44x44px的触摸区域
2. **手势支持**: 支持滑动、缩放等手势
3. **触摸反馈**: 提供触摸反馈效果
