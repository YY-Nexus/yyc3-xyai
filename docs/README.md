## 文档分类说明

### 01-项目规范
包含项目相关的规范文档、模板和标准，确保项目按照统一标准进行开发。

### 02-架构设计
包含系统架构设计文档，详细描述系统的技术架构、模块划分和设计思路。

### 03-实施计划
包含项目实施计划文档，明确开发阶段、任务分配和时间安排。

### 04-优化方案
包含系统优化方案文档，描述性能优化、功能优化和体验优化的具体方案。

### 05-测试验证
包含测试验证文档，描述测试计划、测试用例和测试结果。

### 06-实施总结
包含项目实施总结文档，总结项目实施过程中的经验、成果和问题。

### 07-状态报告
包含项目状态报告文档，描述项目的当前状态、进度和风险。

### 08-功能模块
包含各功能模块的详细文档，描述模块的功能、设计和实现。

### 09-审核报告
包含项目审核报告文档，记录代码审核、架构审核和质量审核的结果。

### 10-开发资源
包含开发资源文档，如设计资源、开发工具和开发指南。

### 11-脚本工具
包含脚本工具文档，描述自动化脚本、工具脚本和辅助脚本。

### 12-归档文档
包含归档文档，存储过时、重复或不再使用的文档。

## 文档命名规范

### 编号规范
- 文档编号格式：`序号-项目名-类型-文档名.md`
- 序号范围：01-99
- 类型分类：
  - 架构类
  - 实施类
  - 计划类
  - 总结类
  - 优化类
  - 规范类
  - 测试类
  - 状态类
  - 定义类
  - 技巧类

### 命名示例
- `14-YYC3-XY-架构类-小语AI浮窗系统全面分析报告.md`
- `20-YYC3-XY-实施类-AI浮窗系统实施计划.md`
- `35-YYC3-XY-总结类-AI浮窗系统优化实施总结.md`

## 文档维护

### 文档更新
- 文档更新后需更新版本号
- 重要变更需记录在变更记录中
- 定期检查文档的准确性和完整性

### 文档审核
- 重要文档需经过审核后发布
- 审核内容包括准确性、完整性和规范性
- 审核通过后方可发布

### 文档归档
- 过时文档需移至归档目录
- 归档文档需保留至少6个月
- 重要归档文档可长期保留
