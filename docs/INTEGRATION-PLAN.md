### 3.3 Winston 日志系统整合

#### 3.3.1 Winston 配置

**来源**：yyc3-xy-03 的日志系统

**整合位置**：`/lib/logger.ts`

**整合内容**：

```typescript
import winston from 'winston'

const logger = winston.createLogger({
  level: process.env.LOG_LEVEL || 'info',
  format: winston.format.combine(
    winston.format.timestamp(),
    winston.format.errors({ stack: true }),
    winston.format.json()
  ),
  defaultMeta: { service: 'yyc3-xy-ai' },
  transports: [
    new winston.transports.File({ filename: 'logs/error.log', level: 'error' }),
    new winston.transports.File({ filename: 'logs/combined.log' }),
  ],
})

if (process.env.NODE_ENV !== 'production') {
  logger.add(new winston.transports.Console({
    format: winston.format.simple()
  }))
}

export default logger
```

**选择原因**：

- Winston 是企业级日志解决方案
- 支持多级别日志（error, warn, info, debug）
- 支持日志持久化和轮转
- yyc3-xy-03 已验证可用

**优化目的**：

- 替换现有的简单日志系统
- 提升日志管理能力
- 提高问题排查效率

**预期效果**：

- 日志管理效率提升 50%
- 问题排查时间缩短 40%
- 日志可读性提升 30%

**实施步骤**：

1. 安装 Winston 依赖：`bun add winston`
2. 创建 `/lib/logger.ts`
3. 配置日志级别和格式
4. 配置日志文件路径
5. 替换所有 `console.log` 为 `logger`
6. 配置日志轮转和清理

**使用示例**：

```typescript
import logger from '@/lib/logger'

logger.info('User login', { userId: '123', ip: '192.168.1.1' })
logger.warn('API rate limit', { endpoint: '/api/chat', count: 100 })
logger.error('Database connection failed', { error: err.stack })
```

**风险与缓解**：

- **风险**：日志切换导致性能下降
- **缓解**：异步日志，避免阻塞主线程

#### 3.3.2 日志级别规范

**日志级别定义**：

| 级别 | 用途 | 示例 |
|------|------|------|
| **error** | 错误级别，需要立即处理 | 数据库连接失败、API 调用失败 |
| **warn** | 警告级别，需要注意但不影响运行 | API 调用慢、内存使用高 |
| **info** | 信息级别，记录关键操作 | 用户登录、API 调用成功 |
| **debug** | 调试级别，仅用于开发调试 | 函数调用、变量值 |

**日志格式规范**：

```typescript
logger.level('message', {
  // 关键字段
  userId: string,
  requestId: string,
  action: string,
  result: string,
  duration: number,
  error?: Error,
  // 自定义字段
  ...data
})
```
